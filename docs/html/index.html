Этот проект создан для тренировки в изучении CI/CD в рамках обучения в МАИ CICD_LAB Описание проекта CICD_LAB — учебный проект на языке C (Си), предназначенный для демонстрации базовых арифметических операций, написания юнит-тестов и настройки процесса CI/CD через GitHub Actions. В проект входят исходные файлы, заголовочные файлы, тесты.

Структура проекта .github/workflows/main.yml — файл конфигурации CI/CD (GitHub Actions).

main.c, main.h — основной модуль с арифметическими функциями.

test_main.c — модуль с юнит-тестами для функций основного файла.

unity.h, unity.c, unity_internals.h — файлы тестового фреймворка Unity для запуска тестов.

README.md — основная документация по проекту.

Сборка и запуск Требования GCC или другой C-компилятор

GitHub Actions (для CI/CD)

Библиотека Unity (входит в проект)

Как собрать проект вручную bash gcc -c main.c -o main.o gcc -c test_main.c -o test_main.o gcc -c unity.c -o unity.o gcc main.o test_main.o unity.o -o test_runner ./test_runner Автоматическая сборка и тестирование Вся сборка и запуск тестов также выполняется автоматически через GitHub Actions (см. .github/workflows/main.yml). CI запускается при каждом коммите в ветки main, development, release*.

Описание функций main.c int add(int a, int b) — возвращает сумму двух чисел.

int sub(int a, int b) — возвращает разность двух чисел.

int mul(int a, int b) — возвращает произведение двух чисел.

int div(int a, int b) — возвращает частное двух чисел (делитель не должен быть 0).

int mod(int a, int b) — возвращает остаток от деления (делитель не должен быть 0).

Все прототипы вынесены в main.h.

test_main.c Модуль юнит-тестов, использующий фреймворк Unity (unity.h):

Проверяет корректность каждой арифметической функции (сложение, умножение, деление, вычитание, остаток).

Тесты автоматически запускаются через CI/CD при каждом изменении кода.

CI/CD Workflow (main.yml) реализует автоматическую проверку кода и тестов при каждом коммите/PR.

Для релизов предусмотрено создание ветки release и автозапуск тестов на ней.

Разработка и поддержка Все изменения фиксируются через коммиты.

Для релизов используются теги (например, v1.0.0).

Документация находится в файле README.md и обновляется при каждой важной правке кода.
